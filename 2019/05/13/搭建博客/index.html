在Linux下搭建博客

May 13, 2019

使用hexo+github来搭建博客

1 . hexo是基于node.js,首先安装nodejs:官网
2.设置node和npm为全局

3.安装git
Ubuntu下使用命令安装
apt-get install git
4.安装hexo
当前面所需要的都配置好后就可以安装hexo
npm install hexo-cli -g

命令执行完毕后就可以使用hexo命令来搭建博客
新建一个文件夹,在此文件中来保存博客
hexo init “博客名”         ps:可以随便起,如果提示未找到命令,要么是
根目录下的_config.yml中出错或者没有将hexo加入全局中

5.hexo 命令
hexo s:开启服务 ps:开启服务后可以locahost:4000进行访问
hexo n:新建博客
hexo d:部署博客
hexo g:生成博客
hexo clean:清除缓存
6.将博客部署到github上
如果本地测试没有问题就可以部署到github上.
a.首先在github上创建一个项目,项目名必须为
github账号名.github.io 如ly700.github.io

    本人使用的是ssh秘钥,免去输入密码,首先生成秘钥
一.打开git bash,输入:
    ssh‐keygen ‐t rsa ‐C "your_email@example.com" //引号输入自己的邮箱地址
    二.进入到.ssh目录,打开'is_rsa_pub',复制里面的内容
    三.打开github,添加秘钥,进入到主页面,复制ssh链接


b.打开博客目录下的_config.yum,找到deploy字段

    deploy:
          type: git
          repository: github地址 //粘贴到这
          branch: master

            ps:此文件中:后面要有一个空格

7.执行命令
npm install hexo-deployer-git –save
然后执行hexo d 部署项目

至此整个博客搭建完毕,就可以访问了 例:(https://ly700.github.io)

问题总结

1.我第一个遇到的问题是hexo s 未找到

是因为我没有将hexo设置为全局,当没有在博客的目录中下输入这段命令就会出现这个问题.

2.当切换hexo主题后执行hexo g命令时layout

首先检查_config.yml文件中是否空格,主题名称是否正确,有没有多一层文件夹等等.实在不行就重新git一下主题
